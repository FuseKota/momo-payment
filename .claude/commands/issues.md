# GitHub Issue 作成

ユーザーの問題報告を元に、関連コードを調査して GitHub Issue を作成する。

## 入力

$ARGUMENTS

## 手順

### 1. 問題の分析

問題の概要から以下を特定:

- 種別: バグ / 機能要望 / 改善 / タスク
- 優先度: 高 / 中 / 低（影響範囲・緊急性から判断）
- キーワード: コンポーネント名、機能名、エンティティ名
- ラベル: 種別 + 優先度を必ず設定（複数可）

#### 優先度の判断基準

| 優先度 | 基準                                           |
| ------ | ---------------------------------------------- |
| 高     | 本番障害、セキュリティ問題、主要機能のブロック |
| 中     | 機能改善、UX向上、開発効率に影響               |
| 低     | 軽微な改善、リファクタリング、ドキュメント更新 |

#### ラベル対応表

| 種別                   | ラベル             |
| ---------------------- | ------------------ |
| バグ                   | `bug`              |
| 機能要望・改善・タスク | `enhancement`      |
| 高優先度               | `priority: high`   |
| 中優先度               | `priority: medium` |
| 低優先度               | `priority: low`    |
| ブロック中             | `pending`          |

### 2. 関連コードの調査

Grep / Glob / Read で関連コードを調査:

1. キーワードでコード検索
2. 関連ファイルを3-5個特定
3. 必要に応じて読み込み、現状把握

### 3. Issue 内容の作成

```markdown
## 概要

[問題の説明]

## 関連ファイル

- `path/to/file1.ts`
- `path/to/file2.tsx`

## 期待される動作

[あるべき姿]

## 現状

[現在の問題点]

## 対応案（任意）

[解決策の提案]
```

### 4. Issue の登録

```bash
gh issue create --title "タイトル" --label "ラベル1,ラベル2" --body "$(cat <<'EOF'
本文
EOF
)"
```

**注意**:

- `--label` オプションで **種別ラベル + 優先度ラベル** を必ず付与すること
- 例: `--label "bug,priority: high"` または `--label "enhancement,priority: medium"`
- 複数ラベルはカンマ区切り

### 5. 結果報告

登録した Issue の URL を報告する。
